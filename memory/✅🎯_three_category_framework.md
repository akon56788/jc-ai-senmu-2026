# 3 つのカテゴリフレームワーク

**設定日**: 2026-05-05  
**適用範囲**: ツール横断（Claude Code・Codex・Cowork・Notion）

---

## 3 つの種別

### 📌 1. 専務のプロジェクト
**定義**: 専務理事対応プロジェクト・期限付き案件  
**進捗管理**: Notion で進捗管理  
**優先度**: 🔴 最高優先度  
**期限**: あり（明確）  
**担当**: Code + Codex + Cowork  
**スレッド命名規則**: `[進捗][性質] [20字以内の本質] — [詳細補足]`
- 進捗アイコン: ✅完了 / 🚀進行中 / ⏳保留中
- 性質アイコン: タスク内容に応じて（📋📊📝等）
- 例: `✅📋 LOM発送準備 — 資料・スケジュール確認`
- 例: `🚀📊 月次レポート — 進捗・予算・人事`

**例**:
- あきる野青年会議所「専務理事対応（AI活用）」プロジェクト
- LOM 月次発送資料管理

---

### 📌 2. 専務のタスク（プロジェクト外含む）
**定義**: プロジェクト傘下以外の単発タスク・業務サポート  
**進捗管理**: Notion で進捗管理  
**優先度**: 🟡 高優先度  
**期限**: あり（タスク依存）  
**担当**: Code + Codex + Cowork  
**スレッド命名規則**: `[進捗][性質] [20字以内の本質] — [詳細補足]`
- 進捗アイコン: ✅完了 / 🚀進行中 / ⏳保留中
- 性質アイコン: タスク内容に応じて（📋📊📝等）
- 例: `✅📝 委員会資料 — 議題・提案内容`
- 例: `⏳📋 週次レポート — LOM進捗・委員会連絡`

**例**:
- 週次・月次レポート作成
- 委員会資料準備
- 一時的な業務サポート

---

### 📌 3. 環境開発・改善系
**定義**: 自分たちの開発環境・ツール・ワークフロー改善  
**進捗管理**: ツール横断（GitHub・Drive・ChatThread）  
**優先度**: 🟢 ベストエフォート・趣味レベル  
**期限**: なし（ベストエフォート）  
**担当**: Code（主導）+ Codex（参照）  
**スレッド命名規則**: `[進捗][システム] [20字以内の本質] — [詳細補足]`
- 進捗アイコン: ✅完了 / 🚀進行中 / ⏳保留中
- システムアイコン: 固定（⚙️🛠️💾🔧📦等）
- 例: `✅⚙️ ツール横断フレーム — 3カテゴリフレームワーク確立`
- 例: `🚀🛠️ GitHub Actions整備 — 自動テスト・デプロイ`

**例**:
- GitHub リポジトリ構成改善
- GitHub Actions テンプレート作成
- Drive ← → GitHub 同期ルール定義
- Claude Code・Codex・Cowork 間の連携最適化
- Notion ワークフロー改善

## スレッド命名規則まとめ

| 種別 | 優先度 | 命名パターン | 例 |
|------|--------|------------|-----|
| **1. 専務プロジェクト** | 🔴 最高 | `⏳[期限]: [内容]` | `⏳5/8期限: リポジトリ整理` |
| **2. 専務タスク** | 🟡 高 | `📋[期限]: [内容]` | `📋毎週月曜: レポート作成` |
| **3. 環境開発** | 🟢 BE | `[性質]: [内容]` | `🚀GitHub Actions 整備` |

---

## Notion 連携方針

- **1. 専務プロジェクト**: Notion で進捗管理（実績・期限・レビュー）
- **2. 専務タスク**: Notion で進捗管理（タスクリスト・チェックリスト）
- **3. 環境開発**: GitHub Issues・チャットスレッドで管理（Notion 不要）

---

## ツール横断での扱い

### 優先度別アクション
1. **種別 1・2** が Notion に登録 → すぐ対応
2. **種別 3** は Chat スレッド相談 → ベストエフォート対応
3. 種別 1 の期限迫っていたら種別 2・3 は一時停止

### スレッド作成時のチェックリスト
- [ ] このスレッドはどの種別に分類されるか？
- [ ] 種別に応じた命名規則を適用したか？
- [ ] 期限がある場合、明記されているか？
- [ ] Notion に登録する必要があるか？（1・2 なら yes）
