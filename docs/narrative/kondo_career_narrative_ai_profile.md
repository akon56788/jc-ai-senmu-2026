## 1. AI向け要約

近藤純史さんは、行動科学を背景に、ITインフラ・クラウド・SaaS・情報システム・運用保守・BPR・PMO・プリセールス・事業開発を横断してきた実務家である。

キャリア全体を通じて一貫しているのは、単一職能の専門家というより、複数の関係者・技術・業務・顧客要望・運用制約を読み解き、全体が動く形へ変換する役割である。

特に以下の傾向が強い。

- 複雑な業務・技術・関係者構造を整理し、運用可能な形にする
- 顧客・営業・開発・保守・運用・サービスデスク・経営層など、異なる言語を持つ関係者の間に立つ
- サービスや事業を、立ち上げ後の運営・改善・拡大フェーズで伸ばす
- 技術そのものよりも、技術が人・組織・顧客体験にどう効くかに関心を持つ
- 公私ともに、社会課題・キャリア支援・組織や集団のあり方への関心を持ち続けている

#33文脈で見ると、この職務経歴は「人間の認知・価値観」と「AI/デジタル基盤」を接続するテーマの前史である。近藤さんはAI以前から、業務・技術・顧客・組織をつなぐ接続レイヤーとして働いてきた。

特徴:

- 多数のステークホルダーと連携し、事業全体のディレクション・推進・統括を担う

- 「上から命令するリーダー」ではなく、「関係者間の力学を読み、接続・調整するリーダー」
- Human-AI協働でも、AIを上下関係で並べるのではなく、得意不得意に応じて職務設計する発想と相性がよい

- 専門を狭く深めるだけでなく、複数領域の境界をまたぐことで価値を出している

## 9. #33 Human-AI Mutual Optimization への接続

### 9.1 人間側モデルとしての職務経歴

近藤さんの職務経歴は、AIが参照すべき「人間側モデル」として以下を示している。

- 全体像を把握してから動きたい
- 関係者の判断軸・制約を理解したい
- 単なる作業ではなく、その作業が事業・組織・人にどう効くかを見たい
- 役割境界が曖昧な状況でも、間をつなぐことで前に進める
- ただし、見えてしまうぶん、自分が抱え込みすぎるリスクがある

### 9.2 AI/デジタル側との接続

AI/PMO運用では、近藤さんの経験は次のように活用できる。

| 近藤さんの経験 | AI/PMO運用での対応 |
|---|---|
| サービスデスク統括 | AIからの問い合わせ/依頼を受ける窓口設計 |
| 運用受入・手順書化 | AI-to-AI handoff、Stop Rules、Issue Intake |
| プリセールス | AI出力をユーザーや関係者に伝わる提案へ変換 |
| 事業/組織マネジメント | 複数AI/ツールの役割分担、責任境界 |
| 情シス改善PMO | GitHub/Drive/Notion/Chatの運用改善 |
| キャリア支援 | 人間側の動機・納得感・違和感を扱う |

### 9.3 研究テーマとしての仮説

職務経歴から見える仮説:

1. 近藤さんは、AI活用を「効率化ツール」ではなく「関係者・知識・業務・価値観を接続する仕組み」として捉えやすい。
2. AI協働が機能するほど、近藤さんの強みである構造化・接続・PMO力は外部化される。
3. 一方で、AIの失敗や摩擦が見えるほど、近藤さんがそれを拾いすぎるリスクもある。
4. したがって、Human-AI Feedback Loop は「AIが賢くなる仕組み」だけではなく、「近藤さんが抱え込みすぎない仕組み」である必要がある。

## 10. AIがこの素材を使うときの指示

### 10.1 タスク説明に使う場合

AI/PMOは、近藤さんにタスクを説明するとき、次を含める。

- このタスクがどの関係者・組織・将来運用に効くか
- 近藤さんのどのコアロールが発揮されるか
- どこまで近藤さんが判断し、どこからAI/PMOが受け持つか
- 抱え込みを防ぐため、完了条件・切り上げ条件を明示する

### 10.2 Issue設計に使う場合

AI/PMOは、Issueを切るとき次を確認する。

- このIssueは実務タスクか、ログストアか、研究素材か
- 近藤さんの違和感を「その場で全部設計」させていないか
- 具体的な成果物があるか
- Low-Opsに反していないか
- 子Issueに分けるべき抽象論を親Issueで抱えていないか

### 10.3 レビュー時に見る観点

AI/PMOは、近藤さんのナラティブに合う結論だけを採用しない。

必ず次を確認する。

- ナラティブと整合しない摩擦はないか
- 「つなぐ力」が、抱え込みに変わっていないか
- AI/PMOが近藤さんの代わりに拾うべき摩擦を、本人に戻していないか
- 抽象設計に逃げず、実際の運用摩擦が1つ減っているか

## 11. LLM用コンパクトプロンプト

以下は、この素材をLLMに渡すときの短縮版である。

```text
近藤純史さんは、行動科学を背景に、ITインフラ、クラウド、SaaS、情報システム改善、運用保守、BPR、PMO、プリセールス、事業開発を横断してきた実務家です。

キャリア全体の一貫したテーマは、複雑な業務・技術・関係者構造を読み解き、人や組織が動ける形に変換することです。

TISではデータセンター、自社クラウドIaaS、サービスデスク、運用設計を経験し、パソナテックではSaaSマーケットプレイス企画やメーカーアライアンス型プリセールスを経験し、SHIFTでは運用保守/BPR/CS事業の組織長、事業開発、情報システム改善コンサル、PMOを担っています。

強みは、水平型リーダーシップ、複数ステークホルダーの判断軸・制約を読む力、業務・技術・事業を接続する力、サービスや運用を仕組みにする力です。

一方で、見えてしまう・拾えてしまうために、本人が抱え込みすぎるリスクがあります。AI/PMOは、近藤さんの構造化力を活かしつつ、違和感や運用摩擦を本人だけに戻さず、Issue・ルール・引き継ぎ・役割分担へ外部化する支援をしてください。
```
