## Key Points

### AI投資

資料では、SHIFTが上期に約25億円規模のAI投資を行い、同規模の受注を獲得していることが示されている。

投資カテゴリには、AIソリューション開発人件費、AI関連システム利用料、AIツール開発人件費、AI人材採用費、イベント・広告宣伝・外注などが含まれる。

これは、近藤さんのJC / 個人PoCが単なる趣味的なAI活用ではなく、AI投資・AIデリバリー・AI変革が会社レベルの主要テーマになっている環境と接続していることを示す。

### AI BPaaSの位置付け

AI BPaaSでは、業務可視化と自動化が中心パターンとして示されている。

関連テーマ:

- 業務の可視化
- 業務フローやマニュアルの分析
- 課題、ROI、AI適用可能性の特定
- 業務分析をもとにしたAIエージェント構築
- AIによる業務自動化
- 人手依存の業務からAIドリブンな業務運用への移行

これはJC / 個人PoCとかなり近い。

```text
JC専務理事業務
-> 会議、議案、Drive/Notion/GitHub、定例作業を可視化
-> ボトルネックと標準化候補を抽出
-> AI PMO / Gemini Ops Lane / Knowledge Pack / Issue運用を構築
-> 人間のルーチン負荷を下げ、次年度引き継ぎ可能な資産を残す
```

### AI BPaaSの事業モデル観点

資料では、独自の分析技術で業務を可視化し、社内開発したエージェント知見を活用してAIで業務を自動化する流れが示されている。

近藤さんの emerging role は、以下のように整理できる。

```text
Multi-vendor AI Deployment Strategist

現実の業務を理解する
-> ボトルネックと制約を整理する
-> AI / モデル / ベンダーごとの役割を切る
-> SSOT、権限境界、ハンドオフ、フォールバックを設計する
-> AIエージェントを実務プロセスへ配置する
-> 生産性、QCD、運用負荷を測定して改善する
```

### 社内AI変革の参照点

資料では、SHIFTが1か月で892業務を可視化し、210人相当の効率化を狙う文脈が示されている。

この参照点は、以下の観点で重要。

- 短期間での業務可視化
- AIエージェント資産の再利用
- 外部展開前の内部業務変革
- 生産性を人数相当・工数相当で測る発想

近藤さんのPoCは小規模だが、同型の学習をしている。

- 専務業務の可視化
- Issue / Task / Context フローの記録
- AI PMOレーンの構築
- 生産性ポストモーテム
- 手運用ベースラインと実作業工数の比較

### 顧客の声との接続

資料では、顧客の声として、SHIFTのAI活用を真似したい、現行業務を大きく変えずに自動化したい、コストセンター業務を経営層に可視化したい、定型業務に多数の自動化対象がある、分析観点が評価された、という趣旨が示されている。

これは近藤さんの議論と強く接続する。

- コストセンターは投資判断が難しい
- AI ROIは高単価・売上直結領域ほど説明しやすい
- 一般事業会社や非営利団体では、制約資源を前提にしたAI導入設計が必要
- チャットAI活用だけでなく、業務プロセス再設計と運用可視化が本丸になる
