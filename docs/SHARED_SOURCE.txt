## 基本的な考え方

このファイル群は、Claude Chat、Cowork、Claude Code、Codex、ChatGPT といった複数の AI ツール間で「**共通の文脈・ルール・前提**」を共有するためのものです。

各ツールはローカルプロジェクト機能を持っており、そのプロジェクトで **このフォルダ（Google Drive）を参照** することで、ツール間での一貫性を保ちながらも、各ツールの得意分野に特化した作業ができるようになっています。

## 基本方針

### 1. Google Drive を SSOT（Single Source of Truth）とする

- コンテキストファイルの本体：Google Drive 内の「専務理事対応（AI活用）」フォルダに配置
- Drive URL: https://drive.google.com/drive/folders/11ryHnY2sXh9Ofio3vSsFHxOQhOr6Wcau
- 各ツールのローカルプロジェクト設定で、Drive フォルダを参照先として指定
- 更新は Drive に集約することで、複製・重複を回避

### 2. ファイル形式：`.txt` 形式で Google Drive の自動変換を防止

- **背景**：Google Drive は `.md` ファイルを自動的に Google Docs (`.gdoc`) に変換するため、AI ツールから plain text として読み込めなくなる
- **対策**：`.txt` 拡張子を使用し、自動変換を防止
- **実効性**：`.txt` 形式のまま Google Drive から参照可能 → AI ツールが plain text context として認識可能

### 3. 読み取り優先度（各ツール共通）

優先度の高い順：

1. **`.txt` files** （Google Drive に配置、自動変換なし）
2. **`.md` files** （ローカルプロジェクト内に保有、Drive 未登録）
3. **`.gdoc` files** （古いファイル、使用禁止、old フォルダに移動済み）

### 更新時の流れ

1. **編集者**がローカルで `.txt` ファイルを修正
2. Drive にアップロード（手動または CLI 経由）
3. 各ツールのプロジェクト設定で参照を確認
4. 変更内容を MEMORY.md に記録（必要に応じて）

### 更新時の注意

- `.txt` 版を優先して更新する
- `.gdoc` 版は旧変換ファイルとして扱い、参照しない
- Drive の `old/` はアーカイブ領域であり、現行コンテキストの配置先にしない
- Code / Codex で Drive Desktop を使う場合は、上記ローカル同期パスから配置・検証できる

## よくある質問

### Q. 古い `.gdoc` ファイルは削除してもいい？

**A.** 一旦 `old/` フォルダに移動した状態で保有してください。1-2 ヶ月様子を見て、参照がなければ完全削除OK。

### Q. `.txt` ファイルを更新したら、全ツールに反映される？

**A.** はい。Google Drive の `.txt` ファイルを更新すれば、各ツール（Chat・Code・Cowork・Codex）のプロジェクト設定で参照しているファイルが自動更新されます。

### Q. ローカルプロジェクトに `.md` ファイルを保有するのはなぜ？

**A.** Git 版( Phase 1 ) への移行に備えるため。ローカルで版管理できるようにしておくと、GitHub 移行時のスイッチが容易になります。

### Q. SHARED_SOURCE.txt は何回読む？

**A.** **毎回読んでください**。新しい質問やタスク開始時に、優先度・ルール・ツール分担が正しく理解されているか確認するためです。

### Q. スレッド命名は誰の責任？

**A.** **Claude Chat（方針層）が提案**します。Cowork・Code・Codex が完了時に「命名提案」を Chat に依頼し、Chat が AGENTS.md.txt の「スレッド命名規則」と「ツール別推奨パターン」に基づいて提案。ユーザーが最終判断。
