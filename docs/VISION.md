## 🎯 プロジェクト趣旨・背景

あきる野青年会議所・専務理事として、プロジェクト型業務と定常業務が混在する環境での「タスク死＆混乱」から脱却するため、IT・業務コンサルの観点から「自分自身の業務をコンサルして作った」システム。

将来的な組織内への横展開や、本業に活かせる PoC にもなることを想定。

---

## 🛠️ 実現技術

各 AI ツールの**プロジェクト機能**を使い、共通のルール・前提・ノウハウを書いた「コンテキストファイル群」を共有するだけ。追加の専用システムや複雑な API 連携は不要。

### 技術スタック

| レイヤー | ツール / 技術 | 役割 |
|---------|-----------|------|
| **AI エージェント** | Claude Chat、Cowork、Code、Codex、ChatGPT | 対話・設計・運用・実装 |
| **コンテキスト共有** | Markdown、Skills、プロジェクト機能 | 全ツール間での前提・ルール共通化 |
| **データベース** | Notion、Google Sheets | 実データ管理・集計 |
| **ファイル管理** | Google Drive | SSOT |
| **MCP 連携** | Google Drive MCP、Notion MCP、Sheets MCP | SaaS との自動同期 |
| **テンプレート** | Word、Excel、Markdown | 業務標準化 |

## 💡 ユースケース

### 対象業務：「混在モデル」

- **プロジェクト型業務** — 期限あり、スコープ固定、成果物固有
- **定常業務** — 繰り返す、ルーチン化（定型/非定型、定期/不定期が混在）

### 実際の運用例

**定常業務（LOM 発送）**：Cowork が自動実行 → テンプレート展開 → Drive アップロード → Sheets 更新

**新規プロジェクト立案**：Chat で議論 → Code でテンプレート生成 → Cowork で Sheets・Notion に登録

---

## 🧠 設計思想

### 1. Token 死の回避
- Chat → 構想・設計に集約
- Code → ドキュメント・テンプレート生成（1回の生成で再利用）
- Cowork → 運用・操作（毎日のルーチンに効率的）

### 2. 「共通ルール」による品質担保
コンテキストファイル群を統一し、使う AI ツールを業務・リテラシー・職務に応じて選択可能にする。ツールが違っても、文脈と品質担保の前提は引き継げる。

### 3. 市民開発の課題を意識
- コンテキストファイル → 組織の「思考テンプレート」
- Notion / Sheets → 組織の「データテンプレート」
- AI ツール → 「テンプレートを動かす手」
