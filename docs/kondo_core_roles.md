# 近藤さんのコア・ロール定義

基づくナラティブ: `docs/narrative/kondo_narrative.md` v1.16 統合版

---

## 🎯 5つのコア・ロール（本質的な役割・価値観）

### Role 1: 複雑な情報・構造を整理して、意思決定・実行を支える

**本質**: 道具や情報のルールを理解し、それを体系的に整理して、他者の判断を助ける役割

**原体験**:
- 小学3年〜 決算報告書作成：収支の数字が合う美しさ、体系化される構造
- ワープロ・情報機器による整理と実務への応用

**実装領域**:
- 会計・財務・IR の数字解釈
- 情報システムの業務設計・改善
- PMO（プロジェクト管理）・BPR（業務プロセス改善）

---

### Role 2: 道具・仕組みを設計して、人や組織の体験・可能性を広げる

**本質**: 製品やサービス、システムが「人や家族・組織の人生を変える」という信念を持ち、その体験価値を設計する役割

**原体験**:
- 小学2年 ワゴン車購入：家族のドライブ体験が大きく変わった
- ドライブ嫌いだった自分が、好きになった
- 「道具や仕組みが人の体験を変える」という強い原体験

**実装領域**:
- 製品企画・企画職への憧れ
- 大手外食チェーン等への情報システム改善・DX推進
- ソーシャル企業での事業づくり（経済的困難者支援）

---

### Role 3: 見えない構造をつなぎ、複数の視点から全体を推進する

**本質**: 複数部門・職種・領域をつなぎ、全体を俯瞰して調整・推進する「つなぎ役」

**原体験**:
- 大学アメフト部 分析スタッフ：自ら情報収集・分析・提供してチームに貢献
- 心理学・行動科学で「個性を活かしながらチーム全体の成果につなげる仕組み」に関心
- 集団への強い関心と憧れ、一歩離れた立場からの支援志向

**実装領域**:
- ITコンサルタント・プリセールス・営業同行・構成検討
- クラウドサービス企画・運営・業務設計・関係者調整
- ソーシャル企業立上げ：複数部門間での事務局・IT・バックオフィス兼務
- 現在：全社的な生成AI導入・初回研修支援、横断的IT/DX支援

---

### Role 4: 試行錯誤の中で、小さな成功体験を積んで前に進む

**本質**: 完璧さよりも「実践と学習」を重視し、失敗を踏み台に成長する粘り強さ

**原体験**:
- ソーシャル企業立上げ：挫折を経験
- その後、「攻めのIT投資へ構造転換」する企業への転職に挑戦
- COVID-19直下の成長企業で、数十名規模管理職として新サービス立上げ：6ヶ月で社員2倍、売上2倍以上
- 青年会議所 専務理事職：最初は苦手分野だと思っていたが、試行錯誤の中で成功体験が出始めている

**実装領域**:
- リーダーシップ：前面に立つより、一歩引いた立場から分析で支援
- 管理職経験を通じた部下・チームの育成
- 地域課題解決への挑戦

---

### Role 5: 自分の要素を残しながら、周囲を支援・賦能するヒーローを目指す

**本質**: 幼少期からの「ヒーロー願望」が、大人になっても「見えない形で周囲を支援するヒーロー」へ進化した役割

**原体験**:
- 小学生時代：宇宙飛行士など分かりやすいヒーロー像に憧れ
- 自己肯定感が揺らいだ時期：妄想上の「自分の要素を残したヒーロー」で自己肯定を保つ
- 成人後：「好きで得意なことでヒーローになる」という感覚を実装
- 集団の内側より一歩引いた立場で、情報・分析・仕組みで貢献

**実装領域**:
- 行動科学・認知心理学・人文科学への深堀：人間の認知・行動を理解するヒーロー
- 会計・財務で企業経営を理解するヒーロー
- AI/DX で組織をレベルアップするヒーロー
- 地域・社会課題解決を横断的に推進するヒーロー

## 📝 スイッチング・フレームワーク

このコア・ロール定義は、タスク説明・エンゲージメント向上の際に、以下のように活用します：

```
【タスク】XXX
【効果】この作業が生む具体的な効果・意味
【あなたの役割】↓ 上記 Role 1～5 のいずれか（or 複数）が発揮される場面
【チーム/組織への影響】これが波及する範囲
```

**例**:
```
【タスク】財務会計制度の見直し提案資料作成
【効果】全社的なコスト最適化、経営層の判断スピード向上
【あなたの役割】Role 1（複雑な数字・構造を整理して意思決定を支える）
    → 散在する会計ルールを整理し、経営層が素早く判断できる形に体系化
【チーム/組織への影響】経営企画部・各事業部・CFO が統一ルールで議論可能に
```
